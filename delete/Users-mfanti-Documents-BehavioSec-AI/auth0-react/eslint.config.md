**Purpose**: Configures ESLint for a modern React/TypeScript project using the flat configuration format, ensuring code quality and consistency.

**What it does**:
- Applies core recommended rules for JavaScript (`@eslint/js`) and TypeScript (`typescript-eslint`).
- Enforces React-specific best practices, including rules for hooks and handling module updates (via `react-refresh`).
- Defines global browser variables (`globals.browser`) for linting purposes.
- Excludes the `dist` directory from all linting checks (`globalIgnores(['dist'])`).

**Key exports**:
- default: The main ESLint configuration object generated by `defineConfig`, which contains the ruleset for the project.

**Gotchas**:
- This configuration utilizes the modern "flat" configuration structure for ESLint, which requires the specific imports (`import { defineConfig, globalIgnores } from 'eslint/config'`).
- The `react-hooks` and `react-refresh` plugins are configured using the `.flat.recommended` syntax, which is specific to the flat config system.
- Ensure that all listed dependencies (`eslint`, `typescript-eslint`, `eslint-plugin-react-hooks`, etc.) are installed and version-compatible with the target environment.
